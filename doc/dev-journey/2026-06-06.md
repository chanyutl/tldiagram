# Development Journey: June 6, 2026

This document chronicles the design iterations, debugging processes, features implemented, and architectural pivots we went through while enhancing the tldraw-based custom link connection overlay.

---

## 1. What We Finally Decided to Do (Implemented Features)

### A. Context Hierarchy Rectification (Bug Fix)
* **Problem**: The connection overlay handles component was throwing the error: `useEditor must be used inside of the <Tldraw /> or <TldrawEditor /> components`.
* **Fix**: Moved the `<ConnectionOverlay />` component inside the `<Tldraw>` element's child tree in [src/App.tsx](file:///home/chanyut/projects/happygocoding/tldiagram/src/App.tsx). This exposed the React context provider instantiated by `Tldraw` to the overlay, resolving the hook error.

### B. Schema Compliance for Tldraw v5 (`richText` Upgrade)
* **Problem**: Programmatic shape creation threw `ValidationError: At shape(type = geo).props.text: Unexpected property`.
* **Fix**: Discovered that `tldraw` v5 replaced the legacy flat `text` string property with a structured `richText` formatting model. Imported `toRichText` from `'tldraw'` and updated all rect, circle, and text shape creation calls to use `richText: toRichText('...')` instead of `text: '...'`.

### C. Signature Alignment for `editor.select`
* **Problem**: Selecting created connections threw `TypeError: Cannot read properties of undefined (reading 'id')` on mouse release.
* **Fix**: Changed the selection call in `ConnectionOverlay.tsx` from `editor.select([currentDrag.arrowId])` to `editor.select(currentDrag.arrowId)` since the API expects individual ID rest parameters (`...ids`) rather than an array.

### D. Dynamic Edge Pin Projection (Cursor-Follow)
* **Problem**: The original design relied on 4 static pinpoint handles (top, right, bottom, left), which felt rigid.
* **Fix**: Implemented a dynamic cursor-tracking projection model. Using `editor.inputs.currentPagePoint` reactively inside the tldraw `track()` state-wrapper, we project the user's cursor onto the nearest outer edge of the rectangle, rendering a single fluid connection pin that follows the pointer along the boundary.

### E. Scaling Handles Priority Conflict Resolution
* **Problem**: When a shape was selected, the connector pin overlay interfered with clicking and dragging the corner scaling handles.
* **Fix**: Added a camera-zoom-aware distance check (`threshold = 20 / editor.getZoomLevel()`). If the pointer moves within 20 screen-pixels of any corner of the shape's page bounds, the connector pin is temporarily hidden. This gives the scaling handles 100% priority.

### F. Text Focus Deactivation
* **Problem**: The linkage handle remained active and visible when typing text labels, causing overlay clutter.
* **Fix**: Added a check using `editor.getEditingShapeId()`. The dynamic pinpoint is completely hidden whenever any shape is actively in text-editing mode.

### G. Shortcut-Triggered Context Menu (`Ctrl + Shift + Click`)
* **Problem**: Right-clicking the canvas to spawn shapes conflicted with the native tldraw right-click context menu.
* **Fix**: Overrode the trigger to `Ctrl + Shift + Left Click` (button 0) on the empty canvas, using event listeners registered in the capture phase to intercept tldraw's default selection handlers and open our custom shape selector.

## 2. What We Did NOT Continue (Pivots & Dropped Features)

### A. Canvas Right-Click Spawner
* **Status**: *Dropped/Pivoted*
* **Reason**: Tldraw manages complex selection-context rendering on right-click. Attempting to suppress it degraded the native editor capabilities. We pivoted to the `Ctrl + Shift + Click` keyboard modifier gesture to spawn our menu cleanly.

### B. Floating Sidebar Panel Redesign
* **Status**: *Dropped Entirely*
* **Reason**: After redesigning the sidebar as a floating obsidian-glassmorphic panel using geometric styling (Syne/Space Mono typography), we evaluated user utility and decided that a sidebar menu was unnecessary for the final simplified canvas interface. We dropped the side panel completely, cleaning up all unused code, actions, and styles to optimize performance and prevent lint errors.
