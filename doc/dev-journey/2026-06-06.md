# Developer Journey: June 6, 2026

## 1. The Quest (Goal)
- **Objective**: Refine and debug a custom arrow connection system on a `tldraw` canvas, ensuring handles are highly interactive, sleek, and do not conflict with native editor tools.
- **Context**: Users need to draw connection links (arrows) between shapes dynamically. Our task was to make the connection pinpoints align fluidly with the user's cursor along the border of shapes, resolve critical layout bugs, make the workspace look extremely clean, and resolve control overlap issues.

## 2. The Wall (Blockers & Errors)
- **Struggle 1 (Context Hook)**: When mounting the overlay sibling to the canvas, it crashed with:
  `Uncaught Error: useEditor must be used inside of the <Tldraw /> or <TldrawEditor /> components`
- **Struggle 2 (Tldraw v5 Schema)**: Creating shapes with a text property crashed the runtime schema validator:
  `ValidationError: At shape(type = geo).props.text: Unexpected property`
- **Struggle 3 (Parameter Mismatch)**: Attempting to select a created connection crashed the editor:
  `TypeError: Cannot read properties of undefined (reading 'id') at Editor.select`
- **Struggle 4 (UI Overlaps)**: 
  - The dynamic edge linkage pins completely blocked pointer events on tldraw's corner handles, making shape resizing impossible.
  - Linkage pins remained active and visible when typing, blocking text visibility and cursor focus during label editing.
  - suppressive canvas right-clicking conflicted directly with tldraw's own context menus.

## 3. The Pivot (Pivots & Dropped Ideas)
- **Pivoted Path (Right-Click Interception)**: We originally designed a custom right-click context menu. However, right-clicking is heavily integrated with tldraw's native context selectors. Suppressing it degraded standard canvas features. 
  - *Pivot*: We moved the spawn gesture to `Ctrl + Shift + Left Click` using capture-phase event listeners to cleanly intercept selection box drawing.
- **Dropped Path (Floating Sidebar Panel)**: We spent time designing a floating obsidian-glassmorphism sidebar using modern Syne & Space Mono fonts. However, when evaluating the minimalist user interface, we decided the sidebar was redundant clutter.
  - *Pivot*: We dropped the sidebar entirely, deleting all unused states and action functions to clean the codebase and satisfy ESLint rules.

## 4. The Eureka (Solution)
- **Breakthrough 1 (Context Fix)**: Moved `<ConnectionOverlay />` directly into `<Tldraw />` children, exposing the internal editor context.
- **Breakthrough 2 (RichText Upgrade)**: Schema validator compliance was achieved by converting flat `text` properties into `richText: toRichText('...')` structures. Corrected `editor.select` call to pass raw ID strings instead of arrays.
- **Breakthrough 3 (Dynamic Edge Clamping)**: Wrote `getClosestPointOnRectangleEdge` to calculate normalized cursor projection coordinates, clamping the pinpoint position to the closest border edge of the unit square.
- **Breakthrough 4 (Zoom-Aware Corners & Focus Hiding)**:
  - Implemented `threshold = 20 / editor.getZoomLevel()`. If the cursor falls within 20 screen-pixels of any corner, the pinpoint hides itself, giving scaling handles highest priority.
  - Read `editor.getEditingShapeId()` to completely disable pinpoint render overlays while editing shape labels.

## 5. Media Hooks (Vlog / Podcast Vibe)
- **Soundbites**:
  - *"Don't fight the canvas context; wrap yourself inside it."*
  - *"Sleek design is as much about knowing what to hide (like connection pins near corners) as it is about what to show."*
- **Visual Highlights**: 
  - Show a screen recording of the pinpoint dot sliding fluidly along a shape's border as the cursor hovers around it.
  - Zoom in on a shape corner to show the pinpoint dot *disappearing* immediately as the cursor gets close to the corner handle, revealing the resizing cursor.
- **Vibe/Energy**: Technical relief. There's a mathematical satisfaction in dividing page boundaries by the camera zoom to make the UI feel pixel-perfect.

# Part 2: Custom Markdown Block & Scroll UX Optimization

## 1. The Quest (Goal)
- **Objective**: Create a custom Markdown block shape for the diagram editor. Users should be able to write rich notes directly on the canvas, styled beautifully, with customizable colors matching the Tldraw color picker.
- **Context**: Standard Tldraw shapes are basic. Rich text/markdown notes allow flexible content like tables, code blocks, and list formats to exist directly on the canvas as fully styled nodes.

## 2. The Wall (Blockers & Errors)
- **Struggle 1 (Type Augmentation)**: Defining the custom shape type caused TypeScript compilation errors where `'markdown'` was not assignable to standard shape types.
- **Struggle 2 (ReadOnly/canEdit)**: Double-clicking did not open the editor container. The shape was not editable on the canvas.
- **Struggle 3 (Missing Connections)**: Linkage arrows could not connect to our new Markdown shape because the overlay connection system only filtered for `'geo'` shapes.
- **Struggle 4 (Scroll Leaks)**: When scrolling the content inside a small Markdown block, the scroll events bubbled up and panned the whole Tldraw canvas, ruining the scroll UX.

## 3. The Pivot (Pivots & Dropped Ideas)
- **Pivoted Path (React Synthetic Event onWheel)**: We tried blocking mouse wheel events with React's `onWheel` synthetic event calling `e.stopPropagation()`.
- **Why We Stopped**: React handles events at the document root (`#root`), while Tldraw registers native event listeners on the canvas. Tldraw's native listeners captured the wheel event before the React delegated listener could intercept and call `stopPropagation`.
  - *Pivot*: Refactored the layout to a separate React component (`MarkdownCard`) and attached a **native DOM listener** directly onto the container element using `useEffect`, halting event bubbling at the target level.

## 4. The Eureka (Solution)
- **Breakthrough 1 (TS Module Augmentation)**: Declared the module `'tldraw'` and augmented the global `TLGlobalShapePropsMap` interface with our `'markdown'` shape schema to fix typing and editor method calls.
- **Breakthrough 2 (canEdit Override)**: Realized Tldraw requires shape utilities to override `canEdit` returning `true` to enable inline double-click editing.
- **Breakthrough 3 (Overlay Filter Extension)**: Expanded `ConnectionOverlay.tsx`'s filter condition from `shape.type === 'geo'` to `(shape.type === 'geo' || shape.type === 'markdown')` so that handles render on our custom shape.
- **Breakthrough 4 (Conditional Scroll Blocking)**: In the native `wheel` handler, checked if the block actually has overflowing content (`scrollHeight > clientHeight || scrollWidth > clientWidth`) or is in edit mode before calling `e.stopPropagation()`. This allows scrolling the card when needed, but pans the canvas when the card does not have scrollable overflow.

## 5. Media Hooks (Vlog / Podcast Vibe)
- **Soundbites**:
  - *"Don't let React's synthetic event bubble run late. Go native to stop Tldraw's canvas scroll."*
  - *"Smart scrolling means only blocking the canvas when your shape actually has text to scroll."*
- **Visual Highlights**:
  - Show a screen recording of scrolling over a non-overflowing Markdown card, panning the canvas, then adding more lines until a scrollbar appears and scrolling *inside* the card natively without moving the canvas.
- **Vibe/Energy**: High satisfaction. Finding the React event delegation leak and solving it natively is a true junior-to-senior eureka moment.
