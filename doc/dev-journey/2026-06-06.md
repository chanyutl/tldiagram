# Developer Journey: June 6, 2026

## 1. The Quest (Goal)
- **Objective**: Refine and debug a custom arrow connection system on a `tldraw` canvas, ensuring handles are highly interactive, sleek, and do not conflict with native editor tools.
- **Context**: Users need to draw connection links (arrows) between shapes dynamically. Our task was to make the connection pinpoints align fluidly with the user's cursor along the border of shapes, resolve critical layout bugs, make the workspace look extremely clean, and resolve control overlap issues.

## 2. The Wall (Blockers & Errors)
- **Struggle 1 (Context Hook)**: When mounting the overlay sibling to the canvas, it crashed with:
  `Uncaught Error: useEditor must be used inside of the <Tldraw /> or <TldrawEditor /> components`
- **Struggle 2 (Tldraw v5 Schema)**: Creating shapes with a text property crashed the runtime schema validator:
  `ValidationError: At shape(type = geo).props.text: Unexpected property`
- **Struggle 3 (Parameter Mismatch)**: Attempting to select a created connection crashed the editor:
  `TypeError: Cannot read properties of undefined (reading 'id') at Editor.select`
- **Struggle 4 (UI Overlaps)**: 
  - The dynamic edge linkage pins completely blocked pointer events on tldraw's corner handles, making shape resizing impossible.
  - Linkage pins remained active and visible when typing, blocking text visibility and cursor focus during label editing.
  - suppressive canvas right-clicking conflicted directly with tldraw's own context menus.

## 3. The Pivot (Pivots & Dropped Ideas)
- **Pivoted Path (Right-Click Interception)**: We originally designed a custom right-click context menu. However, right-clicking is heavily integrated with tldraw's native context selectors. Suppressing it degraded standard canvas features. 
  - *Pivot*: We moved the spawn gesture to `Ctrl + Shift + Left Click` using capture-phase event listeners to cleanly intercept selection box drawing.
- **Dropped Path (Floating Sidebar Panel)**: We spent time designing a floating obsidian-glassmorphism sidebar using modern Syne & Space Mono fonts. However, when evaluating the minimalist user interface, we decided the sidebar was redundant clutter.
  - *Pivot*: We dropped the sidebar entirely, deleting all unused states and action functions to clean the codebase and satisfy ESLint rules.

## 4. The Eureka (Solution)
- **Breakthrough 1 (Context Fix)**: Moved `<ConnectionOverlay />` directly into `<Tldraw />` children, exposing the internal editor context.
- **Breakthrough 2 (RichText Upgrade)**: Schema validator compliance was achieved by converting flat `text` properties into `richText: toRichText('...')` structures. Corrected `editor.select` call to pass raw ID strings instead of arrays.
- **Breakthrough 3 (Dynamic Edge Clamping)**: Wrote `getClosestPointOnRectangleEdge` to calculate normalized cursor projection coordinates, clamping the pinpoint position to the closest border edge of the unit square.
- **Breakthrough 4 (Zoom-Aware Corners & Focus Hiding)**:
  - Implemented `threshold = 20 / editor.getZoomLevel()`. If the cursor falls within 20 screen-pixels of any corner, the pinpoint hides itself, giving scaling handles highest priority.
  - Read `editor.getEditingShapeId()` to completely disable pinpoint render overlays while editing shape labels.
