# Developer Journey: June 8, 2026

## 1. The Quest (Goal)
- **Objective**: Containerize TLDiagram using Docker with Caddy as the web server for automatic SSL/TLS termination via Let's Encrypt.
- **Context**: TLDiagram is a React/Vite SPA that needs to be deployed to a VPS at `chanyutl-tldiagram.happygocoding.com`. The setup must support both local testing (plain HTTP) and production (HTTPS) without duplicating Dockerfiles.

---

## 2. Key Decisions & Rationale

### Multi-stage Dockerfile with `CADDYFILE` build arg
A single `Dockerfile` handles both environments via a build argument:
```dockerfile
ARG CADDYFILE=Caddyfile
COPY ${CADDYFILE} /etc/caddy/Caddyfile
```
- Production uses `Caddyfile` (default, HTTPS + Let's Encrypt)
- Local uses `Caddyfile.local` (plain HTTP on port 80, no TLS)

### Two Compose files — explicit over implicit
- `docker-compose.yml` — production, ports 80/443, named cert volumes
- `docker-compose.local.yml` — local, port 8080, passes `CADDYFILE=Caddyfile.local` build arg

Explicit naming (`docker-compose.local.yml`) was chosen over Docker Compose's implicit override mechanism (`docker-compose.override.yml`) for clarity.

### Port 8080 for local testing
Binding to port 80 on Linux requires root privileges. Port 8080 works without `sudo` and avoids conflicts with other local services. Change the host-side port in `docker-compose.local.yml` if 8080 collides.

### CSP hardening
Audited the production build (`dist/`) before writing the CSP:
- **`script-src`**: Dropped `'unsafe-inline'` — Vite outputs only external module scripts, no inline scripts.
- **`style-src`**: Kept `'unsafe-inline'` — required due to Google Fonts `@import` in the bundled CSS.
- **`connect-src 'self'`**: Confirmed — no external API calls, analytics, or error tracking in the source.
- **Google Fonts domains**: Kept — three font families (Inter, Outfit, Plus Jakarta Sans, Space Mono, Syne) are loaded from `fonts.googleapis.com` and `fonts.gstatic.com`.

### Health check
Added `wget`-based health check (available in `caddy:alpine`) to both compose files so Docker can detect silent failures beyond process crashes.

### Let's Encrypt rate limit warning
The production `Caddyfile` uses the real domain directly. **Do not run `docker compose up` locally** — Caddy will attempt ACME HTTP-01 validation, fail, and burn rate-limit attempts (5 failures/hour per domain).

## 3. Files Created

| File | Purpose |
|---|---|
| `Dockerfile` | Multi-stage build: Node builder → Caddy runner |
| `Caddyfile` | Production: HTTPS, security headers, asset caching |
| `Caddyfile.local` | Local: plain HTTP on `:80`, no TLS |
| `docker-compose.yml` | Production: ports 80/443, cert volumes, health check |
| `docker-compose.local.yml` | Local: port 8080, `Caddyfile.local` build arg |
| `.dockerignore` | Excludes `node_modules`, `dist`, `.git`, `.claude`, `.agents`, `.gemini`, `gh-page`, `doc`, `skills-lock.json` |

## 4. Commands

### Local testing
```bash
docker compose -f docker-compose.local.yml up --build
# Open http://localhost:8080
```

### Production deploy (VPS)
```bash
# Prerequisite: DNS A record for chanyutl-tldiagram.happygocoding.com must point to the server's public IP
# Prerequisite: ports 80 and 443 must be open in the server's firewall

docker compose up --build -d
```

### Update on VPS (manual workflow)
```bash
git pull && docker compose build && docker compose up -d
```

### Verify deployment
```bash
# HTTP → HTTPS redirect
curl -I http://chanyutl-tldiagram.happygocoding.com
# Expected: 301/308 redirect to https://

# SSL cert issued
curl -Iv https://chanyutl-tldiagram.happygocoding.com
# Expected: SSL handshake succeeds, cert from Let's Encrypt/ZeroSSL

# Security headers present
curl -I https://chanyutl-tldiagram.happygocoding.com
# Expected: X-Frame-Options, X-Content-Type-Options, Content-Security-Policy

# SPA routing fallback
curl -I https://chanyutl-tldiagram.happygocoding.com/some-subpath
# Expected: HTTP/2 200 (serves index.html)
```
